02 — Sync

Remote Sync プラグインと S3 バケットで同期する

ここでは Remote Sync（Remotely Save 派生プラグイン）と、自分の S3 バケットを使って Obsidian ボルトを端末間で同期するイメージをまとめます。 実際に運用する際は、必ず公式ドキュメントと README を確認してください。

Plugin

Remote Sync プラグインの概要

Remote Sync は、Remotely Save をベースにセキュリティ面の改良を加えたコミュニティプラグインです。 Amazon S3 や S3 互換ストレージ、Dropbox、OneDrive 個人用、WebDAV など複数のクラウドに対応し、 モバイル版 Obsidian でも動作します。

  • 公式 Obsidian Sync とは別物（サポート対象外）
  • AES-GCM によるエンドツーエンド暗号化オプション
  • 手動／一定間隔の自動同期、保存時同期などに対応

重要なボルトで試す前に、必ずローカルバックアップを取り、 テスト用の小さいボルトで挙動を確認しておくことをおすすめします。

S3 Bucket

S3 バケットを用意する（概要）

AWS S3 を例に、Remote Sync と組み合わせるための最小構成をイメージだけ書いておきます。

  • AWS アカウントを作成し、S3 コンソールで新しいバケットを作成
  • リージョンはすべての端末からアクセスしやすい場所を選択
  • IAM で「そのバケット専用」のユーザーを作成し、アクセスキーを発行
  • バケットポリシーや IAM ポリシーで、必要最低限の権限（読み書き）に絞る

実際にコード例付きで手順を解説している記事（Remotely Save + S3 の例）もあるので、 運用前に最新情報を確認してください。:contentReference[oaicite:6]{index=6}

Setup

Obsidian 側で Remote Sync を設定する

Obsidian デスクトップでの基本的な流れは次のようになります（UI はバージョンによって変わる可能性があります）。

  • Obsidian を開き、歯車アイコンから 設定 を開く
  • コミュニティプラグイン → プラグインを探索 から「Remote Sync」を検索してインストール
  • プラグインを有効化し、「Remote Sync」の設定画面へ
  • クラウドサービスに Amazon S3 / S3 互換 を選択
  • バケット名・リージョン・アクセスキー ID・シークレットキーなどを入力
  • 任意で暗号化パスワードを設定し、テスト接続を実行
  • 問題なければ 初回同期 を実行する

初回同期では、ローカルとリモートのどちらを正とするか慎重に判断してください。 既存バケットに別のデータがある場合は、事前に新しいバケットを切る方が安全です。

Multi-device

端末ごとの同期設定のコツ

2 台以上の端末で同じボルトを使うときの、典型的な手順イメージです。

1 台目（「親」端末）

  • ローカルにボルトを作成し、PARP 構成でフォルダを整える
  • Remote Sync + S3 を設定して 最初のアップロード を実行

2 台目以降

  • Obsidian をインストールし、空のボルト or 新しいフォルダを用意
  • 同じように Remote Sync プラグインをインストール
  • 同じバケット・リージョン・アクセスキー・暗号化パスワードを設定
  • Remote Sync の「同期」または「リモートから取得」を実行して内容をダウンロード

基本的には「どの端末でも同じ S3 バケットを共有し、Remote Sync が仲介してくれる」 というイメージです。実運用では、Wi‑Fi の安定した環境でのテストを強くおすすめします。

Remote Sync / Remotely Save はコミュニティプラグインであり、利用は自己責任で行ってください。
