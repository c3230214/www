03 — Ubuntu

Ubuntu に Obsidian を入れて Remote Sync + S3 で同期する

Ubuntu デスクトップ環境に Obsidian をインストールし、 他の端末と同じように Remote Sync + S3 バケットで同期するまでの流れをざっくり整理します。

Install

Ubuntu に Obsidian をインストールする

もっとも簡単なのは Snap を使う方法です。インターネット接続済みの Ubuntu で、 ターミナルから次のコマンドを実行します： 

sudo snap install obsidian --classic

インストール後は、アプリケーションメニューに Obsidian が追加されます。 AppImage や Flatpak など別の方法もありますが、このサンプルでは Snap を前提にします。

First Run

初回起動とボルト作成

Obsidian を起動したら、最初に次のどちらかを選びます：

  • すでに他の端末で使っているボルトのフォルダを指定する
  • 新しいボルトを作成し、あとから Remote Sync で同期する

このページでは「他の端末と同期する」前提なので、 まだローカルにファイルがない場合は 空のフォルダを作ってボルト化 し、 Remote Sync で S3 から中身を持ってくる想定で進めます。

Remote Sync

Ubuntu でも Remote Sync + S3 を設定する

手順そのものは他の OS と同じで、コミュニティプラグインから Remote Sync を導入し、 S3 バケットの情報を入力します。

  • 設定 → コミュニティプラグイン → プラグインを探索
  • 「Remote Sync」をインストールして有効化
  • Remote Sync の設定で、S3 バケットのリージョン・名前・資格情報を入力
  • 暗号化パスワードを、他の端末と同じものに設定
  • 初回は「リモートからローカルへ」同期してボルトを埋める

Snap 版で権限まわりの問題が出る場合は、公式ドキュメントやフォーラムの情報も確認しつつ、 AppImage / Flatpak など他のインストール方法も検討してください。:contentReference[oaicite:9]{index=9}

Tips

Ubuntu 端末を「安心して同期に参加させる」ための小ネタ

  • 最初はテスト用の小さなボルトで同期の挙動を確認する
  • ネットワークが不安定なときは大きな同期を走らせない
  • VS Code の拡張機能など、ボルト内のファイルを同時に触るツールは注意
  • 重要な変更を加える前に、S3 とローカル両方のバックアップを取っておく

Ubuntu も含めて、どの端末でも「PARP 構成」と「Remote Sync の設定」を揃えておくことで、 マルチデバイス環境でも迷いにくいノート環境を維持しやすくなります。

Snap / AppImage / Flatpak など、Ubuntu でのインストール方法は公式ドキュメントもあわせて確認してください。
